# DeepSeek-V3.1 Release / DeepSeek-V3.1：融合通用与推理的混合模型

## 一句话摘要

DeepSeek-V3.1 于 2025 年 8 月发布，是首个将通用对话能力（V3）和深度推理能力（R1）融合到单一模型中的混合架构，支持"思考模式"和"非思考模式"的自由切换，同时大幅增强了工具使用和智能体（Agent）任务能力。

---

## 完整核心内容翻译

### 1. 核心创新：混合推理架构

DeepSeek-V3.1 代表了一个根本性的范式转变。在此之前，用户需要在通用模型（DeepSeek-V3/Chat）和推理模型（DeepSeek-R1）之间做出选择。V3.1 通过混合架构消除了这一选择——一个模型，两种模式：

- **非思考模式**（deepseek-chat）：快速直接的回答，适合日常对话、简单问题
- **思考模式**（deepseek-reasoner）：启用扩展思维链推理，适合数学、编程、逻辑等复杂问题

模式切换仅需更改聊天模板，无需加载不同的模型权重。

### 2. 模型规格与训练

- **总参数量**：6710 亿（37B 激活）——与 V3 相同的 MoE 架构
- **上下文窗口**：128K token
- **基座训练**：在 DeepSeek-V3 基础上进行了 8400 亿 token 的继续预训练
  - 长上下文扩展阶段 1（32K）：6300 亿 token
  - 长上下文扩展阶段 2（128K）：2090 亿 token
- **精度格式**：FP8 微缩放（Microscaling），为下一代硬件优化

### 3. 性能提升

#### 推理效率

与 DeepSeek-R1-0528 相比，V3.1-Think（思考模式）在推理效率上有显著提升——达到相同答案所需的推理步骤更少，思考过程更加精炼高效。

#### 智能体（Agent）能力

这是 V3.1 最突出的改进之一。通过后训练优化，V3.1 在工具使用和多步骤智能体任务上大幅超越了前代模型：

- **代码智能体基准**：超越 DeepSeek-V3-0324 和 DeepSeek-R1-0528
- **搜索智能体基准**：复杂搜索任务的多步推理能力显著增强
- **SWE 基准**：软件工程任务表现出色
- **Terminal-Bench**：终端操作任务能力提升

#### 多步推理

V3.1 在需要多步推理的复杂搜索任务上展现了更强的能力，能够将问题分解为子任务、逐步执行并综合结果。

### 4. API 与工具增强

- **Anthropic API 格式支持**：兼容 Anthropic 的 API 调用格式
- **严格函数调用**（Beta）：支持结构化的函数调用输出
- **增强的多步智能体任务能力**：更好地支持复杂工具链和工作流

### 5. 分词器与模板更新

V3.1 更新了分词器和聊天模板，新的配置在 Hugging Face 上可获取。这意味着从 V3 迁移到 V3.1 需要更新推理代码中的模板设置。

## 技术要点

1. **单模型双模式**：首次在单一模型架构中统一了通用对话和深度推理能力，通过模板切换实现模式转换
2. **大规模继续预训练**：在 V3 基础上额外训练 8400 亿 token，重点强化长上下文处理能力
3. **智能体能力飞跃**：后训练专门针对工具使用和多步任务优化，使模型具备真正的 Agent 工作流能力
4. **推理效率优化**：思考模式比 R1-0528 更高效，以更少的推理步骤达到相同或更好的结果
5. **FP8 微缩放**：面向下一代硬件的精度格式优化，为更大规模模型铺路

## 深度解读

DeepSeek-V3.1 的发布标志着大模型行业一个重要趋势的确立：**通用能力与推理能力的融合**。

**为什么融合是必然？** 在实际应用中，用户的需求是连续谱：有些问题需要快速直接的回答，有些需要深度思考。让用户在两个模型之间手动选择不仅体验差，还无法处理那些"介于两者之间"的问题。V3.1 的混合架构让模型自身（或通过简单配置）决定何时启用深度思考，这才是面向生产环境的正确方案。

**Agent 能力为何关键？** 2025 年下半年，AI Agent 成为行业最热门的方向。V3.1 在 Agent 任务上的大幅提升表明 DeepSeek 已经将战略重心从"基准测试性能"转向"实际应用价值"。一个能够有效使用工具、执行多步任务的模型，其商业价值远大于一个仅在学术基准上得分高的模型。

**继续预训练的信号**：8400 亿 token 的继续预训练规模令人印象深刻。这表明 DeepSeek 认为，模型的能力提升不仅来自后训练（SFT + RL），持续扩大预训练数据仍然是提升基础能力的关键手段。这与部分行业观点（"预训练已经触及天花板"）形成了有趣的对比。

## 延伸思考

1. **模式切换的自动化**：V3.1 目前通过模板配置切换思考/非思考模式。下一步是否可以让模型自动判断何时需要深度思考？这将带来更自然的用户体验。

2. **Agent 的安全边界**：随着模型获得越来越强的工具使用和多步执行能力，安全性问题变得更加紧迫。一个能够自主执行多步任务的 AI Agent 需要什么样的安全护栏？

3. **对部署架构的影响**：V3.1 的"一个模型两种模式"设计简化了部署——不再需要为通用和推理维护两套推理服务。但这也意味着单个模型需要同时满足两种场景的延迟和吞吐要求，这对推理基础设施提出了新挑战。

4. **竞争格局的变化**：OpenAI 也在探索类似的融合路线（GPT-4o + o1 的统一）。DeepSeek 率先交付了这一方案，是否会迫使其他厂商加速融合进程？
