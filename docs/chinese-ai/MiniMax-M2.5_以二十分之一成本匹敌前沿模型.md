# MiniMax M2.5: Built for Real-World Productivity

## 原标题 + 中文标题
**MiniMax M2.5: Built for Real-World Productivity**
**MiniMax M2.5：为真实世界生产力而生**

## 一句话摘要
MiniMax M2.5 是一个 230B 参数的开源 MoE 模型，每次推理仅激活 10B 参数，通过自研 Forge 强化学习框架训练，在 SWE-Bench Verified 上达到 80.2%（匹敌 Claude Opus 4.6），而 API 调用成本仅为后者的二十分之一。

## 完整核心内容翻译

### 模型发布

2026 年 2 月 11 日，MiniMax 在 Hugging Face 上以修改版 MIT 许可证开源发布了 M2.5 语言模型。这是 MiniMax 旗舰模型系列的最新迭代，相比前代 M2.1，整体效率提升了 **37%**。

### 架构与参数

M2.5 采用混合专家（MoE）架构：
- **总参数量**：230B
- **每次推理激活参数量**：10B
- 这一 23:1 的参数效率比，使其成为当前最"稀疏"的前沿级模型之一

### Forge：智能体原生的强化学习框架

M2.5 的训练核心是 MiniMax 自研的 **Forge** 强化学习框架，其关键特性包括：

- **智能体原生架构**：Forge 是一个"Agent-native RL 框架"，设计了一个中间层来完全解耦底层训练推理引擎和上层智能体，实现了训练与部署的灵活分离
- **约 40 倍训练加速**：通过优化的异步调度和树状结构的样本合并，实现了相比传统方法约 40 倍的训练速度提升
- **CISPO 算法**：采用 CISPO（一种专为 MoE 模型设计的强化学习算法），确保大规模训练中 MoE 模型的稳定性
- **过程奖励机制**：实现端到端的过程级监控，对任务完成时间进行全程优化

### 训练规模

模型在超过 **20 万个真实世界环境**中进行训练，覆盖超过 **10 种编程语言**，包括 Go、C、C++、TypeScript、Rust、Kotlin、Python、Java、JavaScript、PHP、Lua、Dart 和 Ruby。训练周期为两个月。

### 基准测试表现

| 基准测试 | M2.5 成绩 | 说明 |
|---------|----------|------|
| SWE-Bench Verified | **80.2%** | 匹敌 Claude Opus 4.6 |
| Multi-SWE-Bench | **51.3%** | 全球第一 |
| BrowseComp | **76.3%** | 网页浏览理解 |
| VIBE-Pro | 与 Opus 4.5 持平 | 视觉理解 |
| GDPval-MM（办公任务） | **59.0%** 平均胜率 | 综合办公能力 |

**速度表现**：比 M2.1 快 37%，匹配 Claude Opus 4.6 的 22.9 分钟平均任务完成时间。

### 创新：MEWC 基准

MiniMax 创建了一个名为 **MEWC（Microsoft Excel World Championship）** 的独特基准，包含来自 2021-2026 年 Excel 电子竞技赛事主赛区和地区赛区的 **179 道题目**。该基准评估模型理解竞赛级 Excel 电子表格并使用 Excel 工具解题的能力——这是对"真实世界生产力"的务实诠释。

### 定价策略

MiniMax 提供两个完全同能力的版本：
- **M2.5-Lightning**（100 tokens/秒）：输入 $0.3/百万 token，输出 $2.4/百万 token
- **M2.5**（50 tokens/秒）：价格为 Lightning 版的一半

核心卖点：以 100 tokens/秒的速度连续运行模型一小时，成本仅需 **$1**——而竞品模型的同等使用成本高出 10 倍以上。

### 智能体能力亮点

- **规格撰写能力**：模型能够像架构师一样主动分解和规划项目，"先写规格再写代码"
- 在智能体任务中，搜索轮次比 M2.1 减少 **20%**
- 改进的并行工具调用能力
- 长上下文智能体轨迹中的过程级奖励信号实现了精确的信用分配

## 技术要点

1. **极致的参数效率**：230B 总参数中仅激活 10B（4.3%），这一激活比例远低于 DeepSeek-V3 等竞品，证明了更激进的稀疏激活策略在 MoE 架构中的可行性。

2. **Forge 框架的 40 倍加速**：异步调度 + 树状样本合并的工程组合，使得在 20 万真实环境中进行大规模 RL 训练成为可能。这一框架本身可能比模型更有战略价值。

3. **CISPO 算法保障 MoE 训练稳定性**：MoE 模型在 RL 训练中极易出现专家负载不均衡和训练不稳定的问题，CISPO 算法专门针对这一痛点设计。

4. **Multi-SWE-Bench 全球第一的含金量**：Multi-SWE-Bench 评估模型在多语言真实软件工程任务中的表现，51.3% 的成绩表明 M2.5 在跨语言代码理解和修复方面具有显著优势。

5. **$1/小时的成本革命**：将前沿级模型的使用成本从"企业预算"级别拉到"个人开发者"可承受的范围，可能从根本上改变 AI 辅助编程的普及度。

## 深度解读

MiniMax M2.5 的发布传递了一个清晰的信号：**前沿 AI 能力的"去奢侈品化"正在加速**。

从技术路线看，M2.5 选择了"极致稀疏 + 大规模 RL"的组合。仅 10B 的激活参数意味着每次推理的计算量极低，但 230B 的总参数储备确保了模型的知识容量。配合 Forge 框架的 40 倍训练加速，MiniMax 实现了一种"以 RL 质量换参数规模"的策略——用更少的激活参数和更好的训练方法，达到与大量激活参数模型相当的效果。

值得特别关注的是 **MEWC 基准的创建**。在当前 AI 行业普遍追逐"刷榜"的氛围中，MiniMax 选择创建一个以实际办公生产力为导向的评测标准，这反映了其"产品驱动"而非"论文驱动"的企业文化。Excel 电子竞技赛题作为评测素材，既有标准化的评分体系，又能反映真实世界的复杂表格处理需求——这是一个聪明的选择。

从商业角度看，M2.5 的定价策略极具攻击性。$0.3/百万输入 token 的价格，使其成为目前性价比最高的前沿级模型，直接瞄准了那些"想用 Claude 但预算有限"的开发者群体。MiniMax 2025 年全年的海外收入贡献超过 70%，说明这一低价策略在国际市场上已经取得了显著成效。
