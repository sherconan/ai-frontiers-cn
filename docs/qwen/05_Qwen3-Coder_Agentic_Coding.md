# Qwen3-Coder: Agentic Coding in the World

## 原标题 / 中文标题

**原标题**: Qwen3-Coder: Agentic Coding in the World
**中文标题**: Qwen3-Coder：面向真实世界的智能体编程模型

## 一句话摘要

阿里Qwen团队推出Qwen3-Coder，采用480B参数MoE架构（35B激活），通过代码强化学习和长程智能体RL训练，在智能体编程、浏览器操作和工具使用上达到开源模型SOTA，性能比肩Claude Sonnet 4。

## 完整核心内容翻译

### 模型概述

Qwen3-Coder是阿里最新的智能体编程(Agentic Coding)模型。旗舰版本**Qwen3-Coder-480B-A35B-Instruct**采用480B参数的混合专家(MoE)架构，激活参数仅35B。它在以下三个核心维度上达到了开源模型中的最优表现：

- **智能体编程(Agentic Coding)**：如SWE-Bench等真实软件工程任务
- **智能体浏览器操作(Agentic Browser-Use)**：自动操控浏览器完成任务
- **智能体工具使用(Agentic Tool-Use)**：调用各类外部工具

其综合性能"与Claude Sonnet 4相当"——这是对开源模型一个极高的评价基准。

### 技术规格

- **上下文长度**：原生支持256K token，通过YaRN外推可扩展至100万token
- **训练数据**：7.5万亿token，其中代码数据占比70%
- **设计目标**：在增强软件工程能力的同时，保持通用能力和数学能力

### 预训练方法

预训练在三个维度上进行了系统性推进：

**1. Token规模扩展**
训练数据达7.5万亿token，代码数据占比高达70%。这种高比例的代码数据注入确保了模型对编程语言和软件工程模式的深入理解，同时30%的通用数据保证了基础能力不退化。

**2. 上下文扩展**
针对仓库级代码理解和动态数据处理（包括Pull Request等场景）进行了专门优化。256K原生上下文让模型能够理解完整代码库的上下文关系，而不仅仅是单个文件。

**3. 数据质量**
使用Qwen2.5-Coder对训练数据进行合成数据清洗和重写，提升训练素材的整体质量。这是一个"以模型改进模型"的自举(bootstrapping)策略。

### 后训练创新

**代码强化学习(Code RL)**

团队的核心理念是："所有代码任务天然适合基于执行驱动的大规模强化学习。"

具体做法包括：
- 跨多种真实编程任务扩展Code RL
- 通过自动测试用例生成获取奖励信号
- 优化测试用例的多样性，防止过拟合
- 基于代码执行成功率作为训练信号

这一方法有效提升了代码执行成功率，且无需依赖人工标注的奖励模型。

**长程智能体RL(Long-Horizon Agent RL)**

对于SWE-Bench等复杂软件工程工作流，模型需要进行多轮环境交互——阅读代码、理解问题、编写修复、测试验证。为支持这种长程推理：

- 基础设施支持**并行运行20,000个独立环境**，充分利用阿里云的基础设施
- 实现大规模强化学习训练，无需依赖测试时缩放(test-time scaling)
- 模型在训练中直接学习完整的"发现问题→定位代码→实现修复→验证结果"工作流

### 开发者工具与集成

**Qwen Code**：基于Gemini Code适配的命令行工具，针对Qwen3-Coder优化了提示词和函数调用

**集成支持**：
- 兼容OpenAI SDK（通过DashScope）
- 可集成到Claude Code中（提供两种配置方式）
- 支持Cline集成（OpenAI兼容API端点）

**API访问**：通过阿里云模型工作室(Model Studio)提供服务，支持标准OpenAI SDK调用。

## 技术要点

1. **代码RL的天然适配性**：代码任务的可执行性和可测试性，使其成为RL的理想训练场——奖励信号来自代码执行结果，清晰无歧义
2. **20,000并行环境的长程Agent RL**：将RL从简单的单轮任务扩展到多轮软件工程工作流，是Agent训练方法论的重要突破
3. **70%代码数据的预训练配比**：大比例代码数据注入确保了编程领域的深度理解，同时验证了高比例专用数据与通用能力的共存可能
4. **256K到1M的上下文扩展**：通过YaRN外推实现百万token上下文，为仓库级代码理解提供了必要条件
5. **"以模型改进模型"的数据自举**：使用Qwen2.5-Coder清洗训练数据，体现了LLM驱动的数据飞轮策略

## 深度解读

Qwen3-Coder的发布标志着"AI编程"正在从"代码补全"升级为"软件工程"。

**从代码补全到Agent工程师。** 早期的AI编程工具（如Copilot）主要做代码补全和建议。Qwen3-Coder瞄准的是一个更高层次的目标——让AI像一个初级软件工程师一样工作：理解需求、阅读代码库、定位问题、编写修复、运行测试。SWE-Bench就是衡量这种端到端工程能力的基准。

**长程Agent RL是核心技术差异化。** 大多数编程模型的训练仍局限在单文件、单任务的范围内。Qwen3-Coder通过20,000个并行环境进行长程Agent RL，让模型在训练阶段就学会了多步骤、多轮次的工程决策。这不仅需要算法创新，更需要强大的基础设施支撑——这正是阿里云作为后盾的优势所在。

**"比肩Claude Sonnet 4"的含义。** Claude Sonnet在编程任务上一直是业界标杆之一。Qwen3-Coder声称达到相当水平，如果属实，意味着开源模型在高价值应用场景（软件开发）上正在追平闭源模型。这对整个行业的竞争格局有深远影响。

**可集成Claude Code是一个有趣的信号。** Qwen3-Coder的文档中明确提到了与Claude Code的集成方案，这体现了一种务实的生态策略——不是非此即彼，而是让用户在现有工作流中无缝使用Qwen模型。
